# Firecracker Runtime

## Configuration

Firecracker VM JSON generated by `buildConfig()`:

```json
{
  "boot-source": {
    "kernel_image_path": "/path/to/kernel",
    "boot_args": "console=ttyS0 reboot=k panic=1 pci=off root=/dev/vda"
  },
  "drives": [
    {
      "drive_id": "root",
      "path_on_host": "/path/to/rootfs",
      "is_root_device": true
    },
    {
      "drive_id": "tmp",
      "path_on_host": "tmpfs",
      "is_root_device": false
    }
  ],
  "machine-config": {
    "vcpu_count": 2,
    "mem_size_mib": 1024,
    "smt": false
  },
  "vsock": {
    "vsock_id": "0",
    "uds_path": "/tmp/firecracker-<vmId>.socket"
  }
}
```

## Boot Parameters

| Parameter | Value | Purpose |
|-----------|-------|---------|
| console | ttyS0 | Serial console output |
| reboot | k | Kernel reboot (no ACPI) |
| panic | 1 | Panic on fault |
| pci | off | Disable PCI (not needed) |
| root | /dev/vda | Root device (rootfs) |

## Determinism

- **Kernel Path:** Validated via fs.existsSync() before boot
- **Rootfs Path:** Validated via fs.existsSync() before boot
- **JSON Hashing:** buildConfig() returns vmConfigHash = SHA-256(JSON)
- **Mount Points:** /tmp as tmpfs drive (deterministic)
- **Machine Config:** Fixed 2vCPU, 1GB RAM, no SMT

## Isolation

- **Jailer:** UID/GID enforcement (default 1000:1000)
- **Chroot:** Process confined to rootfs
- **vsock:** Unix domain socket communication (no network)

## Lifecycle

1. buildConfig() → JSON + vmConfigHash
2. Jailer exec → chroot + UID/GID
3. spawn('firecracker', ['--config-file', configPath])
4. API socket created (auto-cleanup on exit)
5. vsock listener ready for commands
6. Code sent via vsock → executed in VM
7. Memory/filesystem snapshotted
8. Process killed, sockets cleaned
